\section{Introduction}
\label{sec:einleitung}

LLMs have become a core component of modern software infrastructure. They are
embedded in applications that read emails, summarise documents, browse the web,
and execute actions on behalf of users --- often with minimal human oversight.
This broad integration makes them highly capable, but also exposes them to a
class of attacks that did not exist before they were deployed at this scale.

The threat this paper focuses on is \textit{prompt injection}. The basic
mechanism is that an attacker embeds malicious instructions inside content the
model is supposed to process, and the model follows them instead of the
legitimate ones~\parencite{perez2022ignore, greshake2023indirect}. Because
LLMs have no reliable way to distinguish developer instructions from
attacker-supplied text, injected content can redirect model behaviour, extract
sensitive information, or trigger unintended actions.

The underlying problem is architectural. A human reader can tell the difference
between a quoted string and an actual command, but an LLM just sees tokens. Any
application that feeds untrusted external data into a model is structurally
exposed to this kind of attack~\parencite{liu2023promptinjection}. OWASP
currently lists it as the most significant security risk for LLM-integrated
applications~\parencite{owasp2024llm}.

What distinguishes prompt injection from something like SQL injection is that
there is no syntactic boundary to enforce. The separation between instructions
and data is semantic, which means it cannot be addressed through simple input
sanitisation. The model must interpret both simultaneously, within the same
context window, with no formal distinction between them.

Researchers have proposed a range of defences --- from input filtering to
architectural redesigns --- but recent evaluations consistently find that these
hold up less well than initially reported, particularly against attackers who
adapt to the specific defence being
used~\parencite{yang2025checkpointgcg, pandya2025astra, jia2025critical}. That
gap between claimed and actual robustness is a central concern of this paper.

\medskip
\noindent\textbf{Research Question} \quad
\textit{How effective is the separation of instructions and data as a defence
strategy against prompt injection, and where do the limits of current LLM
prompt injection defences lie?}

\medskip
To answer this question, the paper first covers how LLMs and prompting work,
since the attack only makes sense in the context of how models process
instructions and external content.
