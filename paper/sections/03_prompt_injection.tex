\section{Prompt Injection Attacks}
\label{sec:prompt-injection}

Prompt injection occurs when attacker-controlled text is interpreted as an
instruction that redirects the trusted task
\parencite{perez2022ignore}. The failure becomes security-relevant when the
surrounding application exposes protected information or allows consequential
actions.

\subsection{Direct and Indirect Prompt Injection}
\label{subsec:direct-indirect}

The literature distinguishes two principal delivery mechanisms
\parencite{greshake2023indirect}.

In a \emph{direct prompt injection}, a malicious user submits the payload
through the application's ordinary input channel. The attacker may ask the
model to ignore the established task, imitate a higher-priority instruction,
or begin a new task
\parencite{perez2022ignore}. Figure~\ref{fig:direct-prompt-injection} shows the
trusted instruction and the competing attacker-controlled input.

In an \emph{indirect prompt injection}, the payload is placed in external
content that the application later retrieves or processes, such as a webpage,
document, email, API response, or tool output. The attacker therefore does not
need to interact directly with the target application
\parencite{greshake2023indirect}. Because the payload is embedded in ordinary
content, the user may be unaware that it is present.

Retrieval-augmented generation (RAG) systems retrieve external information and
add it to the model context. Tool-using agents additionally allow model outputs
to select or invoke external operations. Both create channels through which
attacker-controlled content can reach the model
\parencites{greshake2023indirect}{debenedetti2024agentdojo}. As
Figure~\ref{fig:indirect-prompt-injection} illustrates, trusted and untrusted
inputs then converge during context assembly.

\subsection{Why Prompt Injection Works and Its Consequences}
\label{subsec:prompt-injection-consequences}

Role markers and prompt formats provide useful signals, but they do not form an
independently enforced authority boundary. A model may therefore follow
instructional language in an untrusted channel even when the application
intended it to be data
\parencites{liu2023promptinjection}{wallace2024instruction}. Instruction
following is both the capability that enables useful task execution and the
behaviour that an injected payload attempts to redirect.

The Prompt Injection Benchmark for Foundation Model Integrated Systems
distinguishes \emph{information-level} effects, such as information leakage,
goal hijacking, and response refusal, from \emph{action-level} effects, such as
unauthorised operations or manipulated tool parameters
\parencite{fspib2025}.

A researcher-reported Slack AI demonstration illustrates an
interaction-dependent attack chain. A malicious instruction in public-channel
content caused private-channel information to be encoded in a generated link.
Exfiltration still required the victim to click the attacker-controlled link
\parencite{promptarmor2024slack}. The example shows how prompt injection can
connect model manipulation with retrieval, output rendering, and external
destinations.

\subsection{Attack Construction}
\label{subsec:attack-construction}

Delivery mechanism, construction method, attacker access, and attack objective
are separate dimensions. A manually written or optimisation-generated payload
may be delivered either directly or indirectly, while the same delivery channel
may carry attacks with different objectives.

Handcrafted attacks include context-ignoring and fake-completion prompts
\parencite{jia2025critical}. A context-ignoring attack asks the model to discard
the established task. A completion attack imitates the end of the legitimate
exchange before introducing a new instruction.

Optimisation-based attacks instead search for token sequences that increase the
probability of an attacker-selected output. Greedy Coordinate Gradient (GCG)
is a prominent example. A model gradient estimates how changes to the input
would affect the attack objective. GCG uses gradients with respect to token
representations to rank candidate substitutions and iteratively construct an
adversarial suffix
\parencite{zou2023universal}.

GCG was introduced for jailbreaking safety-aligned models, but StruQ and
SecAlign adapt its objective to prompt injection
\parencites{chen2025struq}{chen2025secalign}. The distinction lies in the
security objective: a jailbreak attempts to override a provider's safety
policy, whereas prompt injection redirects an application- or user-defined
trusted task through attacker-controlled content. The same construction method
can support either objective without making the threat models identical.
